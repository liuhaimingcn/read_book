# 两人共读网站 - 详细方案

## 一、核心功能

| 功能 | 描述 |
|------|------|
| **多文件上传** | 支持上传多个 txt 文件，作为书籍列表 |
| **实时同步阅读** | 两人打开同一本书时，阅读进度、当前页同步 |
| **双人确认翻页** | 两人都点击「读完了」后，才切换到下一页 |
| **阅读会话** | 创建/加入阅读房间，通过链接或房间号邀请对方 |

---

## 二、技术架构

### 2.1 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| **前端** | React + Vite | 单页应用，开发体验好 |
| **实时通信** | Socket.io | 房间内进度、翻页、确认状态同步 |
| **后端** | Node.js + Express | 提供 API 和 Socket 服务 |
| **存储** | 内存 / 本地文件 | 初期用内存存储，可后续接入数据库 |
| **部署** | 单机部署 | 前后端可同域或分开部署 |

## 三、功能流程

### 3.1 创建阅读房间

1. 用户 A 进入首页
2. 上传一个或多个 txt 文件
3. 选择要读的书，点击「创建房间」
4. 系统生成房间 ID 和分享链接
5. 用户 A 进入阅读界面

### 3.2 加入阅读房间

1. 用户 B 通过链接或输入房间 ID 加入
2. 进入同一本书的阅读界面
3. 自动同步到当前页和阅读进度

### 3.3 阅读与翻页

1. 两人看到同一页内容
2. 每人有「读完了」按钮
3. 当两人都点击「读完了」后：
   - 自动翻到下一页
   - 两人的「读完了」状态重置
4. 若只有一人点击，则显示「等待对方读完」等提示

### 3.4 页面分页逻辑

- 按字符数或行数分页（如每页约 800–1500 字）
- 分页在服务端完成，前端只负责展示当前页

## 四、数据模型

### 4.1 房间 (Room)

```typescript
{
  id: string;           // 房间ID
  bookId: string;       // 当前阅读的书
  currentPage: number;  // 当前页码
  readerStates: {       // 两人阅读状态
    [socketId]: boolean;  // 是否已点击"读完了"
  };
  createdAt: number;
}
```

### 4.2 书籍 (Book)

```typescript
{
  id: string;
  name: string;         // 文件名
  content: string;      // 全文
  totalPages: number;   // 总页数
  pageSize: number;     // 每页字数
}
```

### 4.3 Socket 事件

| 事件 | 方向 | 说明 |
|------|------|------|
| `join-room` | 客户端→服务端 | 加入房间 |
| `room-joined` | 服务端→客户端 | 加入成功，下发房间状态 |
| `reader-ready` | 客户端→服务端 | 点击「读完了」 |
| `page-turn` | 服务端→客户端 | 翻页（两人都读完时） |
| `sync-state` | 服务端→客户端 | 房间状态同步（进度、谁已读完） |

## 五、页面结构

### 5.1 页面列表

| 页面 | 路由 | 功能 |
|------|------|------|
| 首页 | `/` | 上传/选择书籍、创建房间、加入房间 |
| 阅读页 | `/room/:roomId` | 阅读主界面、当前页、读完了按钮、对方状态 |

### 5.2 首页

- 上传区域：拖拽或点击上传 txt
- 书籍列表：已上传文件列表，可删除
- 创建房间：选一本书 → 创建房间 → 跳转阅读页
- 加入房间：输入房间 ID 或粘贴链接 → 加入 → 跳转阅读页

### 5.3 阅读页

- 顶部：书名、当前页/总页数、房间 ID（可复制）
- 中间：当前页正文
- 底部：
  - 「读完了」按钮
  - 对方状态：「等待中」「已读完」
- 可选：阅读进度条

## 六、实现步骤（分阶段）

### 阶段一：基础框架

1. 初始化前后端项目（Vite + React、Express）
2. 实现 txt 上传与解析
3. 实现分页逻辑（服务端）

### 阶段二：房间与同步

4. 实现房间创建、加入
5. 集成 Socket.io，实现房间内状态同步
6. 实现「读完了」与双人确认翻页

### 阶段三：体验优化

7. 阅读页 UI 优化（字体、行距、夜间模式等）
8. 房间链接复制、分享
9. 断线重连、异常提示

---

## 七、存储与扩展

### 7.1 初期

- 书籍：上传后存内存，服务重启后清空
- 房间：内存存储，重启后清空

### 7.2 后续可选

- 书籍：存文件系统或对象存储
- 房间：Redis 或数据库
- 用户：登录、历史记录、书签

---

## 八、待确认点

1. **人数**：是否固定为 2 人，还是支持 2–N 人共读？
2. **分页方式**：按字数（如 1000 字/页）还是按段落/章节？
3. **房间有效期**：是否需要过期时间，还是长期有效？
4. **部署方式**：本地开发为主，还是需要部署到云服务器？
5. **是否需要用户登录**：匿名即可，还是需要账号体系？
